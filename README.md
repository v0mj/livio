# Livio OS

`Livio OS` is an `Arch Linux`-based gaming distro scaffold built around:

- a `KDE Plasma` live ISO
- a desktop installer icon on the live desktop
- an `online Calamares + pacstrap` install flow
- selectable target desktops during install
- selectable target kernel during install
- selectable GPU driver strategy during install
- a gaming-first package set for the installed system

## Current shape

This repository is a starting point, not a finished distro release.

What is already scaffolded here:

- `archiso` build workflow
- live-session overlay files
- `Calamares` configuration
- kernel chooser with a Livio gaming-kernel default
- desktop chooser
- GPU chooser
- gaming package groups
- branding placeholders for `Livio OS`
- a small `livio-release` package for system identity files
- a generated `linux-livio` package recipe based on Arch's maintained Zen kernel
- a GPU hint helper for the live installer
- a simple `livio-check-system` command for post-install sanity checks

What is intentionally not solved yet:

- a polished theme pack
- offline multi-desktop installs
- signed Livio package/repository metadata

## Why the installer is online

You asked for a live KDE demo ISO, but also for desktop choice in the installer.
The clean way to do that on Arch is to keep the live ISO fixed and use `pacstrap`
plus `Calamares` package selection for the installed system.

That avoids shipping a huge ISO containing every desktop and every gaming package.

## Build requirements

For a full step-by-step guide, read `BUILDING.md`.

You must build the ISO on an `Arch Linux` environment.

Typical options:

- native `Arch Linux`
- an `Arch` VM
- an `Arch` container with the required privileges
- `WSL2` only if you are comfortable handling an Arch userspace there

Required packages on the build machine:

```bash
sudo pacman -S --needed \
  archiso rsync base-devel pacman-contrib python git \
  bc cpio gettext libelf openssl pahole rust rust-bindgen rust-src xxhash zstd \
  extra-cmake-modules libglvnd ninja qt6-tools qt6-translations \
  kcoreaddons kpmcore libpwquality qt6-declarative qt6-svg yaml-cpp
```

## Build

From an Arch environment:

```bash
cd /path/to/linux-distro
./scripts/build-iso.sh
```

By default the build also creates `linux-livio` and `linux-livio-headers`.
That is a real kernel build and can take a long time. For quick ISO plumbing
checks only, disable it explicitly:

```bash
BUILD_LIVIO_KERNEL=0 ./scripts/build-iso.sh
```

From this Windows workspace with Docker Desktop:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build-iso-docker.ps1
```

Artifacts are written to:

- `build/profile`
- `build/work`
- `build/out`

## Validate source files

Before a full ISO build, run the lightweight source checks from Windows:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate-source.ps1
```

This checks the expected Livio files, script wiring, package-list duplicates,
JSON/SVG syntax, and the Calamares sequence.

## Installer flow

The live ISO boots to `KDE Plasma`.
There is an `Install Livio OS` launcher on the desktop.

The installer is configured to offer:

- manual partitioning
- dual-boot / alongside where supported by `Calamares`
- full disk erase
- `btrfs`, `ext4`, `xfs`, `f2fs`
- desktop choice: `KDE`, `GNOME`, `Xfce`
- kernel choice: `linux-livio`, standard Arch `linux`, or LTS-only
- GPU choice: open-source stack, `NVIDIA`, or hybrid
- additional gaming package groups

`Calamares` is built from the vendored package recipe in
`packaging/calamares/PKGBUILD`, copied into the live rootfs during the build,
and installed from that local package inside `customize_airootfs.sh`. This is
necessary because it is not available in the official Arch repositories at the
moment.

`livio-release` is built from `packaging/livio-release/PKGBUILD` and installed
into both the live system and the target system. It ships the Livio identity
templates and applies `/usr/lib/os-release`, `/etc/lsb-release`, console issue
files, and icon identity during package installation.

`linux-livio` is generated by `packaging/linux-livio/prepare-source.sh` from
Arch's official `linux-zen` package recipe, renamed for Livio, and adjusted with
conservative gaming-oriented latency defaults. The generated packages are copied
into a local `livio-local` repository inside the ISO so the target installer can
install `linux-livio` without needing an external Livio mirror. `linux-lts`
remains installed as the recovery kernel.

After installing, run `livio-check-system` inside the installed system for a
quick check of identity, package ownership, NetworkManager, bootloader tools,
and Flatpak availability.

## Gaming defaults

The target install is set up to pull in a gaming-oriented stack by default,
including:

- `Steam`
- `Lutris`
- `GameMode`
- `Gamescope`
- `MangoHud`
- `Wine`
- `Winetricks`
- `Protontricks`
- `Flatpak`

`Heroic` is not added directly from the official Arch repositories in this
scaffold. A helper script is included on the installed system for installing it
through `Flatpak`.
